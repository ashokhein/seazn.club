# v3/06 — Content Editor, Help Centre & Fixture-Format Gallery

Three education surfaces, one doc: the organiser's competition description editor
(intake #18), end-customer feature documentation (#22), and fixture-format explanation
with mind-map-style diagrams (#21, #23).

## 1. Principle

Seazn's format engine is its moat, and it is currently *explained nowhere*. Every hour of
explanation shipped as product (editor previews, format diagrams, help pages) converts
directly into fewer support messages and better trial conversion. Build all three on
**Markdown/MDX in-repo** — no external tool, versioned with code, free, SEO-owned.

## 2. Competition description editor (intake #18)

- **Storage: Markdown** (portable, renders on public page + exports + emails). No HTML
  blobs, no proprietary JSON.
- **Editor: TipTap** (headless ProseMirror, MIT) configured to emit/ingest Markdown, with
  a deliberately small block set: h2/h3, bold/italic, lists, links, image, blockquote,
  divider, and a **sponsor/CTA button block** (custom node → renders as branded button on
  the public page).
- **Write / Preview tabs** (#18's explicit ask): Preview renders through the *same*
  component the public site uses (`<CompetitionProse>`), so preview = truth, including
  the org's branding.
- Images upload to the existing Supabase storage path (club-logo pipeline reuse), max
  2MB, served resized.
- Sanitisation server-side on save (rehype-sanitize allowlist); description length cap
  20k chars.
- Same editor reused for: division descriptions, org "about" on public site, and (later)
  custom pages from the Jul3 deferred list (sponsors/regulations pages).

## 3. Help centre `/help` (intake #22)

- **In-repo MDX** under `apps/web/content/help/**` → static routes `/help/[...slug]`,
  ISR. Recommendation over Mintlify/GitBook/Notion: nothing to sync, free forever,
  indexable under our domain, and `Tips` (v3/03 §4) deep-link into it (`helpSlug`).
- Structure = the product's own hierarchy, one article per flow, written for the
  non-technical organiser (write from the user's side of the screen — no engine jargon):

```
help/
  getting-started/ (create org → comp → division → entrants → generate → start → score)
  formats/         (one page per format — shares components with §4 gallery)
  registration/    (open registration, custom questions, fees, waitlist, moderation, refs)
  scoring/         (per-sport scoring pages, undo, device links, scorer role)
  schedule/        (board, constraints in plain words, locking, undo)
  sharing/         (visibility in plain words, dashboard, slideshow, embeds)
  billing/         (plans, event pass, trials, receipts, cancel)
```

- Each article: ≤600 words, one hero screenshot or diagram, "common questions" footer.
  Screenshots generated by a Playwright script against seeded demo data
  (`scripts/help-shots.ts`) so they regenerate after UI changes instead of rotting.
- Search: client-side (FlexSearch over a build-time index) — no service.
- Footer/marketing nav gains "Help"; console `?` menu → help + contact.

## 4. Fixture-format gallery with mind-maps (intake #21, #23)

The format picker is where non-experts stall ("what's a stepladder?"). Give every format
an **explainer card + flow diagram + live example**:

- **Diagrams:** hand-authored SVG per format family (league, groups+KO, swiss, KO,
  double-elim, stepladder, americano, ladder) in the mind-map style the founder asked
  for — nodes = stages/pools, arrows = progression feeds:

```
            ┌─ Group A (RR ×5) ─┐  top 2 ─┐
  16 teams ─┤                   │         ├─→ Quarter-finals → SF → Final
            └─ Group B (RR ×5) ─┘  top 2 ─┘           (single elimination)
```

- **Live example:** each explainer embeds a real generated preview via the existing
  `POST /api/v1/format-preview` with a canned 8-entrant set — "here is exactly what you
  get" — rendered with the same bracket/round components as the console.
- **Surfaced in three places:** `/help/formats/*`; the console format picker ("How this
  works →" opens the explainer in a side panel — organisers choose formats confidently
  without leaving the flow); marketing site `/formats` (SEO: "round robin generator",
  "americano tournament format" are high-intent queries).
- Picker also gains a **recommendation strip**: given entrant count + courts + time
  budget, rank 2–3 formats with the trade-off in one sentence each (pure function over
  engine metadata; the AI-prose version is v3/10 backlog).

## 5. Acceptance sketch

Editor round-trips Markdown (golden: save → reload → identical AST); preview HTML ===
public-page HTML for same input; XSS corpus sanitised. `/help` builds statically, search
finds "waitlist"; every format has explainer+diagram+live preview and the picker links to
it. Screenshot script regenerates all images headlessly.
